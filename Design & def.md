 # 2. Definición de casos de uso.

 Los casos de uso ayudan a comprender cómo interactuarán los usuarios con el software y a definir los principales escenarios de uso.

Un prompt de ejemplo podría ser:

Eres un analista de software experto. Estoy construyendo un sistema de reserva de vuelos online. Enumera y describe brevemente los casos de uso más importantes a implementar para lograr una funcionalidad básica


Representa estos casos de uso en el tipo de diagrama más adecuado usando el formato plantUML. Diferencia entre usuarios visitantes y usuarios logueados. Acorde a la sintaxis y buenas prácticas UML, define y describe lo que sea necesario. 


# 3. Modelado de datos:

Entender las entidades que van a jugar un papel relevante, cómo se relacionan y qué información debemos almacenar de cada una es un paso fundamental en la definición del sistema. 


Un prompt de ejemplo podría ser:

Eres un arquitecto de software experto. Cuales son las 3 entidades de modelo de datos esenciales en un sistema de reserva de vuelos online? Dame algunos campos esenciales de cada una y cómo se relacionan
